# Лабораторная работа №7: TCP-сервер (Вариант 18)

## Описание задания
Реализовать серверное приложение, принимающее строку от клиента и возвращающее её в соответствии с вариантом 18.

**Вариант 18: Эхо-цензор**
- Сервер заменяет все гласные буквы на символ '*'
- Гласные буквы: a, e, i, o, u, y (и их заглавные версии A, E, I, O, U, Y)
- Пример: "Hello World" → "H*ll* W*rld"

## Структура проекта

```
lab7_variant18/
```

## Сборка проекта

Для сборки проекта требуется Maven и Java 17+:

```bash
cd lab7_variant18
mvn clean package
```

## Запуск сервера

Запуск сервера на порту по умолчанию (8018):
```bash
java -cp target/lab7-echo-censor-1.0.jar com.student.net.CensorServer
```

Или с указанием своего порта:
```bash
java -cp target/lab7-echo-censor-1.0.jar com.student.net.CensorServer 9090
```

## Запуск клиента

Подключение к серверу на localhost:
```bash
java -cp target/lab7-echo-censor-1.0.jar com.student.net.CensorClient
```

Подключение к удаленному серверу:
```bash
java -cp target/lab7-echo-censor-1.0.jar com.student.net.CensorClient 192.168.1.100 8018
```

## Запуск сканера сети

Сканирование локальной сети 192.168.1.x:
```bash
java -cp target/lab7-echo-censor-1.0.jar com.student.net.NetworkScanner
```

Сканирование другой подсети:
```bash
java -cp target/lab7-echo-censor-1.0.jar com.student.net.NetworkScanner 10.0.0. 8018
```

```

========================================

Вы: Hello World
Ответ сервера: H*ll* W*rld

Вы: Java Programming
Ответ сервера: J*v* Pr*gr*mm*ng
```

## Технические особенности

1. **CensorServer**:
   - Многопоточная обработка клиентов (каждый клиент в отдельном потоке)
   - Автоматическое закрытие ресурсов через try-with-resources
   - Поддержка UTF-8 кодировки
   - Graceful shutdown через хук завершения JVM

2. **CensorClient**:
   - Интерактивный режим работы
   - Поддержка команд выхода (exit, quit, выход)
   - Обработка ошибок подключения

3. **NetworkScanner**:
   - Асинхронная проверка портов с таймаутом
   - Тестирование найденных серверов тестовым сообщением
   - Пропуск собственного IP-адреса

## Вариант 18 - детали реализации

Алгоритм цензурирования:
1. Проход по каждому символу входной строки
2. Проверка символа на принадлежность к гласным (регистронезависимая)
3. Замена гласных на '*', сохранение остальных символов
4. Возврат обработанной строки клиенту

Поддерживаемые гласные: `a, e, i, o, u, y, A, E, I, O, U, Y`
